# ⚠️ СРОЧНО: Настройка базы данных

## Проблема
Без переменной `DATABASE_URL` база данных PostgreSQL не работает, и:
- ❌ Заявки не сохраняются
- ❌ Ресурсы не создаются
- ❌ Ресурсы не загружаются
- ❌ Ничего не работает!

## Решение: Настройте DATABASE_URL прямо сейчас

### Вариант 1: Neon (Бесплатно, 5 минут)

1. **Создайте проект:**
   - Откройте https://neon.tech
   - Войдите через GitHub
   - Нажмите **Create Project**
   - Выберите регион (EU или US)
   - План: **Free**
   - Нажмите **Create Project**

2. **Скопируйте Connection String:**
   - В проекте Neon: **Dashboard**
   - Найдите **Connection Details**
   - Нажмите **Copy** рядом с connection string
   - Строка выглядит так: `postgresql://user:pass@host/db?sslmode=require`

3. **Добавьте в Vercel:**
   - Откройте https://vercel.com → ваш проект
   - **Settings** → **Environment Variables**
   - **Add New**:
     - Name: `DATABASE_URL`
     - Value: вставьте скопированную строку
     - Environment: ✅ Production, ✅ Preview, ✅ Development
   - **Save**

4. **Перезапустите деплой:**
   - **Deployments** → последний деплой → **...** → **Redeploy**

### Вариант 2: Supabase (Бесплатно, альтернатива)

1. Откройте https://supabase.com
2. Создайте проект
3. **Settings** → **Database** → **Connection string** → **URI**
4. Скопируйте строку
5. Добавьте в Vercel как `DATABASE_URL` (см. шаг 3 выше)

## После настройки

1. **Проверьте логи:**
   - Functions → `/api/resources/submit`
   - Должны быть: `✅ Connected to PostgreSQL via pg`

2. **Отправьте тестовую заявку:**
   - Заполните форму
   - Отправьте на модерацию
   - Проверьте логи

3. **Одобрите заявку:**
   - Войдите как админ
   - Одобрите заявку
   - Проверьте, что ресурс создается

## Важно!

- ⚠️ Без `DATABASE_URL` ничего не работает
- ⚠️ После добавления переменной ОБЯЗАТЕЛЬНО перезапустите деплой
- ⚠️ Не делитесь connection string публично
