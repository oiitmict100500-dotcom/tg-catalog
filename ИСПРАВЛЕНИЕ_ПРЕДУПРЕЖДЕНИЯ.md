# Исправление предупреждения url.parse()

## Проблема

В логах Vercel появляется предупреждение:
```
(node:4) [DEP0169] DeprecationWarning: `url.parse()` behavior is not standardized and prone to errors that have security implications. Use the WHATWG URL API instead.
```

## Объяснение

Это предупреждение возникает внутри библиотеки `pg` (PostgreSQL клиент), когда она парсит строку подключения к базе данных. Это **не критично** и не влияет на работу приложения - это просто предупреждение о том, что библиотека использует устаревший метод внутри себя.

## Решение (рекомендуется)

### Вариант 1: Подавить через переменную окружения (самый простой)

1. Перейдите в **Vercel Dashboard** → ваш проект
2. Откройте **Settings** → **Environment Variables**
3. Добавьте новую переменную:
   - **Key**: `NODE_OPTIONS`
   - **Value**: `--no-deprecation`
   - **Environments**: Выберите все (Production, Preview, Development)
4. Сохраните и перезапустите деплой

Это подавит все deprecation warnings в логах.

### Вариант 2: Игнорировать (тоже нормально)

Это предупреждение не критично, можно просто игнорировать. Оно не влияет на функциональность приложения.

## Альтернатива

Код для подавления уже добавлен в `api/db.js`, но переменная окружения работает более надежно.
