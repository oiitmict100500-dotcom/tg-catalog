# Настройка API Backend для Vercel

## Проблема
На Vercel развернут только frontend, а backend не развернут. Поэтому все запросы к `/api/*` возвращают 404.

## Решение

### Вариант 1: Использовать внешний backend (рекомендуется)

1. Разверните backend на отдельном хостинге:
   - Heroku
   - Railway
   - Render
   - DigitalOcean
   - Или любой другой хостинг с поддержкой Node.js

2. В Vercel Dashboard добавьте переменную окружения:
   - **Имя**: `VITE_API_URL`
   - **Значение**: URL вашего backend (например: `https://your-backend.herokuapp.com`)
   - **Environment**: Production, Preview, Development

3. Пересоберите проект на Vercel

### Вариант 2: Использовать Vercel Serverless Functions

Если backend небольшой, можно создать serverless functions на Vercel.

## Текущая конфигурация

Код уже настроен для использования переменной окружения `VITE_API_URL`.

Если переменная не задана, используется относительный путь `/api/*` (для локальной разработки).

## Проверка

После настройки:
1. Откройте консоль браузера (F12)
2. Проверьте, что запросы идут на правильный URL
3. Должны загружаться категории и ресурсы
