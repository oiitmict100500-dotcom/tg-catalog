# Инструкция: как настроить VITE_API_URL в Vercel (по вашему скриншоту)

## Вы сейчас находитесь здесь:

На скриншоте видно, что вы в разделе:
- **Settings** (верхняя навигация)
- **General** (левая боковая панель)

## Что нужно сделать:

### Шаг 1: Перейдите в Environment Variables

1. Посмотрите на **левую боковую панель** (слева от основного контента)
2. Найдите в списке пункт **"Environment Variables"**
3. Нажмите на **"Environment Variables"**

### Шаг 2: Найдите или создайте VITE_API_URL

После перехода в Environment Variables вы увидите список переменных окружения.

1. **Если переменная `VITE_API_URL` уже есть:**
   - Найдите ее в списке
   - Нажмите на нее (или на иконку редактирования)
   - Перейдите к Шагу 3

2. **Если переменной нет:**
   - Нажмите кнопку **"Add New"** или **"Add"**
   - Перейдите к Шагу 3

### Шаг 3: Установите правильное значение

1. В поле **"Name"** (или **"Key"**):
   - Введите: `VITE_API_URL`

2. В поле **"Value"**:
   - **УДАЛИТЕ** текущее значение (если есть)
   - **ВВЕДИТЕ** URL вашего backend:
     - Если backend на Railway: `https://ваш-проект.railway.app`
     - Если backend на Render: `https://ваш-проект.onrender.com`
     - Если backend на Vercel (другой проект): `https://ваш-backend-проект.vercel.app`
   - ⚠️ Обязательно добавьте `https://` в начале!
   - ⚠️ БЕЗ слеша `/` в конце!

3. В разделе **"Environment"** (или **"Environments"**):
   - Выберите все три галочки:
     - ☑ Production
     - ☑ Preview
     - ☑ Development

4. Нажмите **"Save"** (Сохранить)

### Шаг 4: Подождите и проверьте

1. Vercel автоматически перезапустит ваш сайт
2. Подождите 1-2 минуты
3. Обновите страницу сайта (F5)
4. Откройте консоль браузера (F12)
5. **НЕ должно быть** ошибки: "VITE_API_URL указывает на frontend домен!"

---

## Важно!

- **НЕ используйте** URL вида `https://tg-catalog-*.vercel.app` - это frontend!
- **Используйте** URL вашего backend (Railway, Render, или другой проект Vercel)

---

## Если у вас нет backend URL

Если вы не знаете, какой URL использовать, значит backend еще не развернут. См. файл `КАК_НАЙТИ_BACKEND_НА_VERCEL.md` для инструкций по развертыванию backend.
